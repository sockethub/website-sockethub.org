A protocol gateway for the web

View on GitHub Read Documentation

Sockethub is a protocol gateway for the web, a translation layer that enables web applications to communicate with protocols and services traditionally inaccessible from in-browser JavaScript.

By using ActivityStreams as a standardized message format, Sockethub acts as an intelligent proxy server that maintains state and connects to networks, sockets, and endpoints that would otherwise be restricted in browser environments. This approach gives applications and users greater autonomy, privacy, and protocol independence.

    Sockethub 5.0.0-alpha.6

    Sockethub 5.0.0-alpha.6 has been released! This alpha release has major rewrite and architectural improvements across the board, working toward a stable 5.0.0 release.

    • Greatly improved unit and integration tests
    • A New Metadata Platform!
    • An Examples package
    • Migrate to BullMQ
    • Migrate to Bun
    • Improved documentation
    • So much more...

    For a full list of changes, see the Sockethub release page.

    Sockethub 4.0.0

    Sockethub 4.0.0 has been released! This release has a few major updates to the underlying system, namely switching from node-simple-xmpp to xmpp.js, from Kue to Bull and upgrading to socket.io v4. Additionally, as we performed these significant overhauls of the code, we added more tests and optimizations. This release should be overall a lot faster. For a full list of changes, see the Sockethub release page.

    Nick Jennings
